autopilot

URLs open in the right browser profile, automatically.

Route GitHub to your work profile, AWS to your dev account. Set rules once and forget about it.

Chrome Web Store View on GitHub

How it works

1

Install the extension

Add autopilot to each browser profile from the Chrome Web Store.

2

Install the helper

Run one command in Terminal to set up the native bridge.

3

Set your rules

Match by domain or keyword. URLs automatically open in the right profile.

Install the helper

After installing the extension, run this in Terminal:

curl -fsSL https://dantemoon1.github.io/autopilot/install | bash

Works on macOS. No admin required.

Features

Domain rules

Route entire domains like github.com to a specific profile, with optional subdomain matching.

Keyword matching

Match URLs containing aws, teams, or any keyword — great for long auth redirect URLs.

Shared config

Rules are shared across profiles. Configure once, works everywhere.

Multi-browser

Works with Chrome, Brave, Edge, Arc, and Helium. Route URLs across any browser and profile.
